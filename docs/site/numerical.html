Numerical Solver & Settings

The core engine leverages FEniCSx and PETSc to solve the highly non-linear coupled system of equations. This section details the numerical implementation specifics.

1. Staggered Iterative Scheme

The coupled system \(( \mathbf{u}, c, \phi )\) is solved using an alternate minimization (staggered) approach rather than a monolithic Jacobian. This ensures stability in the damage field evolution.

  1. Displacement Step: Solve for \(\mathbf{u}\) at fixed \(\phi, c\).
  2. Diffusion Step: Solve for \(c\) at fixed \(\mathbf{u}, \phi\).
  3. Phase-Field Step: Solve for \(\phi\) at fixed \(\mathbf{u}, c\).

The cycle repeats until the \(\phi\) residual satisfies: \(\| \phi_{n+1} - \phi_n \| < \epsilon \).

2. PETSc Solver Configuration

The suite uses high-performance linear and non-linear solvers tuned for large 3D meshes.

Displacement Solver (Linear)

  • Type: Conjugate Gradient (CG)
  • PC: Geometric Multigrid (GMG) or Algebraic Multigrid (GAMG)
  • Tolerance: \(10^{-8}\)

Phase-Field Solver (Inequality Constrained)

The damage field is irreversibility-constrained (\(\dot{\phi} \geq 0\)). We solve this using a variational inequality solver:

petsc_options = {
    "snes_type": "vinewtonrsls",
    "snes_vi_monitor": "",
    "ksp_type": "preonly",
    "pc_type": "lu"
}

3. Mesh Refinement Strategy

Accurate phase-field modeling requires an element size \(h < \ell / 2\). The suite implements Adaptive Box Refinement:

  • Base Mesh: Coarse everywhere to reduce MPI load.
  • Refinement Zone: A localized box ahead of the notch where \(h \approx \ell / 5\).
  • Transition: Smooth GRADIENT transition to prevent numerical stress reflections.
Warning Setting the length scale \(\ell\) too small relative to the mesh size will cause the crack to "hop" between nodes or fail to propagate.
