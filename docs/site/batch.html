Batch Manager & DOE

The **Batch Manager** is a powerful tool for Design of Experiments (DOE) and parametric sensitivity analysis. It allows you to automate hundreds of simulations and aggregate the results into intuitive charts.

1. Defining a Grid Search

A Grid Search (Full Factorial Design) runs every possible combination of specified parameters. To set one up:

  1. Switch to the Batch Manager tab in the main interface.
  2. Check the boxes for the parameters you wish to sweep (e.g., \(G_c\), \(E\), \(\chi\)).
  3. For each enabled parameter, input values separated by commas (e.g., 2.5, 3.0, 3.5).
  4. Set the Parallel Limit (number of concurrent MPI processes).

2. Parallel Execution

The manager uses a Process Pool to keep your system responsive. While simulations run in parallel, each simulation itself can utilize multiple cores if MPI is configured correctly.

Resource Tip For 3D simulations, it is often more efficient to run 4 simulations sequentially with 4 cores each, rather than 4 simulations in parallel with 1 core each, depending on your CPU's memory bandwidth.

3. Automated Sensitivity Mapping

Once the batch finishes, the suite performs two automated analysis steps:

  • Result Aggregation: Peak loads and fatigue cycles are extracted from each .csv log.
  • Sensitivity Analysis Plotting: A trend chart is generated showing how the target metric (e.g., Peak Load) varies with the independent parameter.

4. Data Structure

Every batch generates a unique timestamped folder:

results/batch_20240410_123000/
├── config.yaml          # Record of sweep params
├── summary.csv          # Aggregated results for DOE
├── sensitivity_plot.png  # Generated trend chart
└── exp_001/             # Individual run data
    └── crack_progress.xdmf
